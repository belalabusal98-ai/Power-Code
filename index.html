A simple checklist to help you build your first app from a SharePoint list.

Important Rule (Very Important):

• If your button does ONLY ONE action, write the formula WITHOUT a semicolon
SubmitForm(MyForm)

• If your button does MORE THAN ONE action, separate each action using a semicolon ;
SubmitForm(MyForm); Notify("Saved", NotificationType.Success)

Power Apps Definitions

Screen App Control Properties Formula Bar Data Source Connector Table / List Record Field / Column Form Gallery Label Text Input Button Navigate OnSelect DisplayMode Save Publish Version Share Environment Power Apps Studio Maker User Preview Tree View Insert Menu

Screen — A container in Power Apps that holds controls and user interface elements, similar to a page in an app.

App — The complete Power Apps project that contains screens, data connections, logic, and user interactions.

Control — An individual element such as a button, label, icon, gallery, or text input used to build the app interface.

Properties — Settings that define how a control looks and behaves, such as Text, Color, Visible, or OnSelect.

Formula Bar — The area where you write Power Apps expressions and logic, similar to writing formulas in Excel.

Data Source — The service where your app reads and stores data, such as SharePoint, Dataverse, Excel, or SQL.

Connector — A bridge that allows Power Apps to connect to another service like Outlook, Teams, SharePoint, or databases.

Table / List — A structured collection of data records and columns coming from a data source.

Record — One single row of data inside a table or list.

Field / Column — A single piece of data inside a record, such as Name, Date, Status, or Email.

Form — A control that allows users to view, create, or edit data from a data source.

Gallery — A control that displays multiple records in a list, grid, or card layout.

Label — A control used to display text or information to the user.

Text Input — A control that allows users to type text or numbers into the app.

Button — A control that triggers an action when selected, such as saving data or navigating to another screen.

Navigate — A function used to move the user from one screen to another inside the app.

OnSelect — A property that defines what happens when a control (like a button) is clicked.

DisplayMode — Controls whether a control is editable, view-only, or disabled.

Save — Stores your current changes but does not make them live for users.

Publish — The process of saving and promoting your app so end users can access the latest version.

Version — A snapshot of your app at a specific point in time; Power Apps keeps a history of versions.

Share — Gives other users permission to run or edit your app.

Environment — A workspace in Power Platform used to separate apps and data, such as Development, Test, or Production.

Power Apps Studio — The design interface where you build, edit, and test your app.

Maker — A person who builds and edits apps in Power Apps.

User — An end user who runs the app but does not edit it.

Preview — Runs the app inside Power Apps Studio to test how it behaves before publishing.

Tree View — A panel that shows all screens and controls in a structured list.

Insert Menu — The menu where you add new controls, layouts, icons, and media to your app.

Hand Notes — Creating Your First Power App

  1. Before opening Power Apps, make sure you already have a SharePoint list ready with the columns you need, such as Title, Description, or Status. Preparing the data first makes building the app much easier. Use clear column names and avoid special characters or starting column names with numbers.
  2. Go to make.powerapps.com and create a new Canvas App from blank. Choose a phone or tablet layout based on how users will access the app, then give the app a clear and recognizable name.
  3. Connect the app to your SharePoint list by opening the Data panel and adding the list as a data source. If the list does not appear, check that you selected the correct SharePoint site.
  4. Insert a Gallery control and set its Items property to your SharePoint list. This will display existing records in a scrollable list that users can select from.
  5. Add a Form control and connect it to the same SharePoint list. Use NewForm to create new records and EditForm to update existing ones. The form will save data automatically when SubmitForm is used.
  6. Insert buttons for common actions such as Submit, Home, and Back. Use SubmitForm to save data, Navigate to move between screens, and Back to return to the previous screen. Keep button labels simple and clear.
  7. Use the Play button to test the app. Try adding, editing, and navigating between records, and adjust the layout and spacing if something does not look right.
  8. Save your app frequently, but remember that saving only stores changes for you. Users will not see updates until you publish the app.
  9. Publish the app after every important change so the latest version is available online. If users say they cannot see updates, it usually means the app was not published.
  10. Share the app with users or groups and give them User access so they can use the app without editing it.
